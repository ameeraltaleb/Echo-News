# Contributing to Echo-News

Thank you for considering contributing to the Echo-News project! We welcome contributions from everyone. Below are some guidelines for getting started and participating in this project.

## Getting Started
1. **Fork the repository**: Click the fork button on the top right of the repository page.
2. **Clone your fork**: Run `git clone https://github.com/your-username/Echo-News.git` in your terminal.
3. **Set up the upstream remote**: Run `git remote add upstream https://github.com/ameeraltaleb/Echo-News.git`.

## Development Setup
1. Make sure you have [Node.js](https://nodejs.org/) installed on your machine.
2. Navigate to the project directory: `cd Echo-News`.
3. Install the required dependencies: `npm install`.

## Running the Project
To start the project, run the following command:
```
npm run dev
```
This will start the development server and open the application in your default web browser.

## Building for Production
To build the project for production, use the following command:
```
npm run build
```
This will create an optimized build in the `dist` directory.

## Code Quality Standards
We use ESLint for linting our JavaScript code. Make sure to check your code with:
```
npm run lint
```
Before submitting your changes, please ensure that all linter errors are fixed.

## Pull Request Guidelines
1. Create a new branch from `main` for your changes.
2. Make sure your branch is up to date with the `main` branch.
3. Write a clear and descriptive commit message.
4. Submit a pull request detailing the changes you've made and why.

## Code Style Requirements
We follow the Prettier code style. Make sure to format your code before submitting:
```
npm run format
```

## Issue Reporting Guidelines
If you encounter any issues, please report them in the `Issues` tab. Provide a clear description of the issue, including steps to reproduce, screenshots, and any error messages you received.

Thank you for contributing to Echo-News! Your support makes a difference!
